# In production

The frontend no longer has a running process. vite build compiles everything into frontend/dist/ вҖ” a folder of static files (index.html, assets/main-abc123.js, assets/main-abc123.css). These are just files on disk.

Nginx does two jobs:

Serves the static frontend files directly from disk вҖ” fast, no Python involved
Reverse proxies any request starting with /api/ to Uvicorn running locally
Uvicorn is never exposed to the internet вҖ” only Nginx talks to it. This is important for security and performance.

## Full request lifecycle in production

### Loading the app (first visit):
1. Browser вҶ’ GET https://thefraynews.com/
2. Nginx reads frontend/dist/index.html from disk вҶ’ responds
3. Browser parses HTML, requests main.js
4. Nginx reads frontend/dist/assets/main.js from disk вҶ’ responds
5. React app boots in the browser

### User logs in:
1. React вҶ’ POST https://thefraynews.com/api/v1/login  { email, password }
2. Nginx sees /api/* вҶ’ proxy_pass to 127.0.0.1:8000
3. Uvicorn вҶ’ FastAPI вҶ’ user_service.login() вҶ’ DB query вҶ’ JWT
4. Response travels back through Nginx to browser
5. React stores the token, sets auth state

### User loads their feed:
1. React вҶ’ GET /api/v1/feed/for-you  (Authorization: Bearer <token>)
2. Nginx вҶ’ Uvicorn вҶ’ get_current_user() decodes JWT
3. hybrid_recommend_articles() runs in memory
4. DB query for article rows
5. JSON response вҶ’ Nginx вҶ’ browser вҶ’ React renders cards
With Docker (the clean way to deploy this)

You'd have two containers orchestrated with Docker Compose:
1. The app (Uvicorn backend server)
2. The database (PostgreSQL database)

services:
  app:
    build: .
    restart: unless-stopped
    depends_on:
      db:
        condition: service_healthy # waits for the healthcheck above to pass
    ports:
      # Exposed to the OS so Nginx on the host can proxy to it
      - "8000:8000"
    environment:
      DATABASE_URL: postgresql://thefray:${DB_PASSWORD}@db:5432/thefray_dev
      DB_PASSWORD: ${DB_PASSWORD}
      SECRET_KEY: ${SECRET_KEY}
      ALGORITHM: HS256
      NEWSAPI_KEY: ${NEWSAPI_KEY}
      NEWSAPI_URL: https://newsapi.org/v2/everything
      CORS_ORIGINS: ${CORS_ORIGINS}
      API_PREFIX: /api/v1
      ACCESS_TOKEN_EXPIRE_MINUTES: 30
      ENVIRONMENT: production
    volumes:
      # Static files (avatars) persist across container rebuilds
      - static_files:/app/static

  db:
    image: postgres:16
    restart: unless-stopped
    environment:
      POSTGRES_USER: thefray
      POSTGRES_PASSWORD: ${DB_PASSWORD}
      POSTGRES_DB: thefray_dev
    volumes:
      # Named volume вҖ” data persists on the host even if the container is removed
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      # App container won't start until Postgres is ready to accept connections
      test: ["CMD-SHELL", "pg_isready -U thefray -d thefray_dev"]
      interval: 5s
      timeout: 5s
      retries: 5

## Only two things are always running on the server:
	1. Nginx вҖ” handles every inbound connection. For static files (JS, CSS, HTML) it reads from disk and responds in microseconds.
	   Python is never involved. For anything under /api/ or /health it forwards the request internally to Uvicorn.

	2. Uvicorn вҖ” runs your FastAPI app. It is never directly reachable from the internet. Only Nginx can talk to it.
	   This is intentional вҖ” Nginx handles SSL, compression, connection limits, and slow clients so Uvicorn only deals with clean, fast requests.


## A full user session traced through every layer:

1. USER OPENS THE APP
   Browser вҶ’ GET https://thefraynews.com/
   Nginx reads frontend/dist/index.html from disk
   Browser receives bare HTML shell + <script src="/assets/main.js">
   Browser вҶ’ GET /assets/main.js
   Nginx reads from disk вҶ’ browser downloads React bundle
   React boots, renders login page

2. USER LOGS IN
   React вҶ’ POST /api/v1/login  { email, password }
   Nginx вҶ’ proxy_pass вҶ’ Uvicorn
   FastAPI: SlowAPIMiddleware checks rate limit (5/min per IP)
   FastAPI: UserService.login() вҶ’ UserRepository вҶ’ SQL query вҶ’ PostgreSQL
   FastAPI: bcrypt verify password вҶ’ create JWT
   Response: { access_token, user } + Set-Cookie: access_token=...
   React stores token in memory, stores user in state

3. USER LOADS THEIR FEED
   React вҶ’ GET /api/v1/feed/for-you  Authorization: Bearer <token>
   Nginx вҶ’ Uvicorn
   FastAPI: get_current_user() decodes JWT вҶ’ UserRepository вҶ’ load user
   FastAPI: get_model_registry() returns app.state.models (already in RAM)
   FastAPI: hybrid_recommend_articles()
     вҶ’ InteractionRepository вҶ’ load user's likes/dislikes/views
     вҶ’ build_user_vector() вҖ” weighted average of TF-IDF vectors (pure math, no DB)
     вҶ’ cosine_similarity() вҖ” score all articles (pure math, no DB)
     вҶ’ KNN.kneighbors() вҖ” find similar articles (pure math, no DB)
     вҶ’ merge scores, filter seen articles, return top 20 IDs
   FastAPI: ArticleRepository вҶ’ fetch 20 Article rows from PostgreSQL
   FastAPI: Pydantic serializes to JSON
   React renders article cards

4. USER LIKES AN ARTICLE
   React вҶ’ POST /api/v1/interactions  { article_id: 42, type: "like" }
   Nginx вҶ’ Uvicorn
   FastAPI: auth check вҶ’ article exists check вҶ’ UniqueConstraint check
   FastAPI: InteractionRepository вҶ’ INSERT into interactions
   Response: 201 Created
   (next feed load will now include this signal in the recommendation)

5. INFRASTRUCTURE MONITORS HEALTH
   Load balancer вҶ’ GET /health         (every 30s)
   Nginx вҶ’ Uvicorn
   FastAPI: returns { status: ok, uptime_seconds: 3600 }

   Uptime monitor вҶ’ GET /health/ready  (every 60s)
   FastAPI: SELECT 1 вҶ’ PostgreSQL (proves DB is reachable)
   FastAPI: models.is_ready check
   Returns 200 if healthy, 503 if DB or models are down
The training pipeline (runs separately, not on every request)
