---------- Scalability Path ----------

PostgreSQL     → Drop-in swap via DATABASE_URL + pip install psycopg2-binary
               → Your config.py + SQLAlchemy setup already supports it

Redis caching  → Cache hybrid_recommend_articles() results per user_id
               → Invalidate on new interaction (TTL ~5 min is fine)

Background     → Move /admin/ingest to a scheduled job (APScheduler or Celery)
ingestion        instead of an HTTP endpoint triggered manually

Containerize   → Dockerfile + docker-compose with postgres + backend services
               → config.py + .env pattern is already 12-factor friendly

Model serving  → At scale, move ML inference out of the FastAPI process
               → Dedicated inference service (e.g. TorchServe or a simple FastAPI sidecar)


Phase 1: PostgreSQL + Alembic
Do these together — they're tightly coupled. Switching to Postgres without Alembic means you're immediately managing schema by hand, which is painful. One session:

database.py — connection pooling for Postgres
Add psycopg2-binary + alembic
Set up Alembic, generate initial migration from your existing models
Swap DATABASE_URL in .env
Why first: Everything else builds on the database. Redis caches DB results. Docker bundles the DB. The scheduler writes to the DB. Get this right before adding complexity on top.

Phase 2: Scheduler (APScheduler)
Now that Postgres is stable, add the scheduler. It's purely Python — no new infra. Easy to test locally.

Phase 3: Docker
Containerize after the stack is stable locally. Your docker-compose.yml will define three services: api (FastAPI), db (Postgres), redis (Redis). Do this before Redis in code because:

It gives you a local Postgres + Redis to develop against
From this point on, docker compose up is your dev environment
Phase 4: Redis caching
With Redis running in Docker, wire up fastapi-cache2 or redis-py to cache the heavy read endpoints (/feed/latest, /feed/topics/{topic}, /feed/for-you). These are your hottest paths and will get hammered in production.

Phase 5: Nginx
Last layer. Add an nginx service to docker-compose.yml as a reverse proxy in front of FastAPI. Handles SSL termination, static file serving (avatars), and request buffering. Only meaningful once everything else is containerized.

Summary

PostgreSQL + Alembic  →  Scheduler  →  Docker  →  Redis  →  Nginx
     (foundation)         (feature)   (infra)    (perf)   (prod edge)
